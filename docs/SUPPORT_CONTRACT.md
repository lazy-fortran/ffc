# ffc Support Contract

This document is the public contract for the active `ffc` compiler path.
A program is supported only when every construct it uses appears in
"Supported now". Everything else is unsupported until the linked issue is
closed with executable tests.

## Architecture boundary

- FortFront owns lexing, parsing, AST storage, semantic analysis, type
  inference, diagnostics, and the compiler-facing frontend query APIs.
- `ffc` owns lowering from FortFront semantic data to LIRIC, the runtime
  ABI, diagnostics produced during lowering, object emission, executable
  emission, and behavioural tests for every claimed feature.
- LIRIC owns native code generation behind its session C API.
- New `ffc` compiler work uses LIRIC session C APIs through
  `iso_c_binding`.
- New `ffc` compiler work does not use LLVM bindings, MLIR bindings,
  HLFIR, or text-IR compiler paths.

## Supported now

| Area | Supported contract |
| --- | --- |
| Input | Standard Fortran source parsed by FortFront's compiler-facing API. Lazy Fortran and Infer-mode inputs are accepted only when FortFront produces the same supported semantic constructs. |
| Output | Native executable or object file emitted by LIRIC. |
| Program unit | One main program plus contained procedures. |
| Scalars | Scalar `integer`, `real`, and `logical` declarations and assignments. Compound declarations (`integer :: a, b, c`). Scalar `parameter` named constants for `integer`, `real` (incl. `real(8)`/`real(dp)`), `logical`, and assumed-length `character(len=*)` with a compile-time literal initializer; the constant folds into uses in expressions, conditions, and `print`. Array, derived-type, and non-literal-expression parameters stay unsupported. |
| Enums | Program-scope `enum, bind(c) ... end enum` blocks. Each `enumerator` binds an integer named constant of default kind: value is the explicit initializer (signed decimal literal), else the previous enumerator plus one, starting at zero. Multiple names per `enumerator` line and several enum blocks per program are supported; the constants fold into expressions, `print`, and `select case`. Module-scope enums and non-literal initializers stay unsupported. |
| Characters | Character literals, fixed-length `character(len=N)` variables, deferred-length `character(len=:), allocatable` with assignment, concatenation including self-aliasing and three-way concat, and compile-time `//` folding for character literal chains. |
| Expressions | Integer and real literals, scalar variable references, integer and real arithmetic, integer comparisons, scalar logical conditions. |
| Termination | Falling off `program main` returns zero. `stop <integer expression>` returns the integer expression as process status. |
| `if` | Integer comparison `if` blocks with terminating branches or branches that assign mergeable scalar values; scalar logical `if (flag)` conditions. Array-element writes inside `if` branches are recognised even when the FortFront semantic flag is missing. |
| `do` | Counted `do` loops with integer bounds and literal integer step (positive or negative). |
| `goto` / labels | Simple `goto <label>` to a statement label in the same main program, including the arithmetic `if (e) l1, l2, l3` form that FortFront desugars to labelled `goto`s. A body that carries any statement label is lowered label-aware: each labelled statement starts a basic block, a `goto` branches to it, and every scalar integer/logical local is held in a stack slot so its value survives arbitrary branch edges (the backend rebuilds SSA). Computed `goto (l1, l2), e` and goto across a contained procedure boundary are not supported. |
| `pause` | `pause` and `pause <message>` continue execution. The interactive halt and stderr banner that gfortran emits are not reproduced. |
| `select case` | Single-arm or multi-arm `SELECT CASE` with literal integer labels, ranges, character labels, multi-label arms (`case (1, 5)`), and `case default`. Arms may terminate (`stop`/`return`) or fall through to a merge after the construct; non-terminated arms carry mergeable scalar values via an N-way PHI. |
| `where` | Single-statement `where (mask) a = expr` and the construct form with a final unmasked `elsewhere`, over fixed-size rank-1 integer, `real` (kind 4), and `real(8)` arrays. The mask is a scalar comparison whose operands are conforming array elements or broadcast scalars (`a > 0`, `b == 0`); integer masks use `==`, `/=`, `<`, `<=`, `>`, `>=`, real masks use `<`, `<=`, `>`, `>=`. Each body assignment lowers per element: the mask element selects between the body value and the current element (the `elsewhere` body selects the complement). Body right-hand sides accept whole-array `+`, `-`, `*`, scalar broadcast, and the elemental real intrinsics that lower to libm (`sqrt`, `exp`, `log`, `sin`, `cos`, `tan`, `abs`). Masked `elsewhere`, nested `where`, and array-section operands are rejected with a diagnostic. |
| `forall` | Single-statement `forall (i = lo:hi[:stride]) a(i) = expr` and the block form `forall (...) ... end forall`, over integer loop indices. Each index lowers to a counted loop; the indices form a sequential loop nest, so a multi-index `forall (i = ..., j = ...)` and a `forall` body that itself contains a `forall` both lower as nested loops. An optional trailing mask (`forall (i = lo:hi, i > 2) ...`) guards the body with a conditional branch; the mask is a scalar comparison over the loop indices and broadcast scalars. Body assignments write rank-1 or rank-2 array elements directly to memory. Masks whose operands are array elements (`a(i) > 0`) are rejected with a diagnostic. |
| `associate` | `associate (name => expr)` over scalar integer, real, or logical selectors, with one or several associations in one construct (`associate (u => a, v => b)`). Each name binds to the selector value for the construct body; the binding is dropped at `end associate`. The body may read the name in expressions, `print`, `stop`, and assignments. Array, derived-type, and character selectors are not yet supported. |
| Procedures | Contained integer, real, and logical functions and subroutines, including early `return` inside each. `stop` inside a contained procedure terminates the program via libc `exit`. Scalar procedure actual arguments use pointer parameters with copy-back for variable actuals. `optional` integer dummies are supported with `present(arg)`: an omitted actual is passed as a null pointer (the caller pads to the callee's parameter count) and `present` is a null check. `if (...)` without an `else` and `.not.` in a condition are supported. A `character(len=*), intent(in)` dummy receives a character actual (deferred-length variable or literal) through a {data pointer, length} descriptor, so `len`/`len_trim` see the actual length. A `character(len=:), allocatable, intent(out)` dummy receives the address of the actual variable's own descriptor, so an assignment inside the callee writes the new data and length back to the caller. |
| Intrinsics | **Numeric:** scalar `abs`, `min`, `max`, `mod`, `modulo`, `sign`, `dim` for integers; `abs`, `min`, `max` for reals; `real()` (integer-to-f32), `dble()` (any-to-f64), `int()` (truncate toward zero), `nint()` (round half away), `floor()`, `ceiling()` as integer results. These lower inline with LIRIC scalar ops and PHI merges. **Math:** real transcendentals `sqrt`, `exp`, `log`, `log10`, `sin`, `cos`, `tan`, `asin`, `acos`, `atan`, `atan2`, `sinh`, `cosh`, `tanh`, `asinh`, `acosh`, `atanh`, `erf`, `erfc`, `gamma` (`tgamma`), `log_gamma` (`lgamma`), and `hypot` lower to the matching libm call on `real(8)` operands; integer actuals promote to real first. **Bit:** `iand`, `ior`, `ieor`, `not`, `ishft`, `ishftc`, `ibits`, `btest`, `mvbits` on integer values. **Array:** `size`, `sum`, `product`, `maxval`, `minval`, `dot_product`, `matmul`, `reshape`, `transpose`, `lbound(a, dim)`, `ubound(a, dim)`, `shape` (array target), `count(mask)`, `any(mask)`, `all(mask)` on fixed-size integer arrays; `lbound`/`ubound` return the declared bound as a compile-time constant. **Character:** `len`, `len_trim`, `trim`, `adjustl`, `adjustr`, `index`, `scan`, `verify`, `repeat`, `achar`, `iachar`. **Command line:** `command_argument_count()` and `get_command_argument(i, value)` in the main program (no optional `length`/`status`). **Iso_c_binding:** `c_loc(x)`, `c_associated(p)`, `c_f_pointer(p, fptr)` on `c_ptr` scalars. **Kind inquiry:** `selected_int_kind(r)` and `selected_real_kind(p[, r])` are constant-folded to the gfortran kind number at lowering time; arguments must be compile-time integer expressions. |
| `print` | Scalar `print *, expr` for integer expressions, real values, character literals, character variables, logical literals, real variables, and logical variables. A bare whole `integer`/`real` array may appear as a print item alongside others (`print *, "tag", arr`); each element prints inline with list-directed spacing. List-directed output matches gfortran byte-for-byte, including `real(8)` G-format (fixed and exponential forms, `Infinity`/`NaN`), `integer(4)` width, `T`/`F` logicals, and the inter-item separator rules. Every Fortran `real` is lowered as `real(8)`. Formatted `print fmt, items` with a literal format string is supported for a single `I`/`A` edit descriptor (`I0`, `Iw`, `A`, `Aw`) applied to each item, and for compound literal formats made from `I`, `X`, `F`, and `A` descriptors on stdout (`I5,2X,F8.3`, `A,A,I5`, `A8,I3`). The `A`/`Aw` descriptor in a compound format prints a character literal, fixed-length character variable, or deferred-length character expression. Other compound descriptors are rejected. Internal `write (buf, fmt) value` into a fixed-length character variable is supported for the same single edit descriptors, blank-padded to the buffer length (single value; no write-to-file-unit). Internal `read (buf, fmt) value` parses a single integer (`I0`/`Iw`) from a character variable via `sscanf` (no real/character read, no read-from-file). |
| Arrays | Fixed-size one- and two-dimensional arrays with compile-time bounds for `integer`, `real` (kind 4), `real(8)`, and `logical` element types. Logical arrays occupy `integer(4)` slots; element write/read, scalar broadcast, whole-array copy, and `print` (`T`/`F`) work for rank-1 and rank-2. Scalar integer parameters and explicit `lower:upper` bounds are supported. Rank-1 arrays support element assignment, element reads, `print`, `stop`, counted-loop subscripts, whole-array assignment from an array constructor (`a = [e1, e2, ...]`, element count must match the declared size), including typed constructors (`[integer :: ...]` truncates real literals toward zero, `[real :: ...]` promotes integer literals) and integer/real implied-do constructors (`[(i*i, i=1, n)]`, bounds and step fold at compile time), whole-array assignment and elemental `+`, `-`, `*` between conforming arrays, and scalar `size`, `sum`, `product`, `maxval`, `minval`, `dot_product`, `lbound(a, dim)`, `ubound(a, dim)`, `count(mask)`, `any(mask)`, `all(mask)`. Rank-1 and rank-2 array sections with compile-time integer bounds are accepted as rvalues in standalone list-directed `print` and whole-array copy/arithmetic when shapes conform. Rank-2 arrays support declaration and scalar element access through `a(i,j)` for reads, writes, `print`, `stop`, argument passing, whole-array copy, whole-array elemental `+`, `-`, `*` over conforming shapes, scalar `size`, `sum`, `product`, `maxval`, `minval`, `matmul`, `transpose`, and `reshape`; `shape(a)` can be assigned to a rank-1 integer array target. Runtime bounds checks are not emitted. An `integer, allocatable :: a(:)` declaration lowers to a zero-initialised 24-byte descriptor; `allocate(a(N))` and `deallocate(a)` are supported (single variable, one dimension, default lower bound). Scalar element access on an allocated 1-D integer allocatable is supported for reads and writes (`a(i)` as an rvalue and as an assignment target, including inside counted `do` loops), computing the address from the descriptor at runtime. Whole-array assignment from an array constructor (`a = [e1, e2, ...]`) is supported for 1-D integer allocatables: the old allocation is freed, fresh storage is allocated to match the constructor length, and elements are stored in order (auto-reallocation). Whole-array assignment between two allocatables and rank-2 allocatables are not yet supported. |
| Pointers | Scalar integer `pointer` and `target` (#245 slice B3a). A `target` local lives in a stable memory slot. Pointer assignment `p => t` makes the pointer address that slot; reads and writes through `p` dereference it, so a write through `p` mutates the target and a later write to the target is seen through `p`. `associated(p)` (one-argument form) and `associated(p, t)` (two-argument form, #245 B3c) report straight-line association state; `nullify(p)` disassociates. The two-argument form compares the compile-time address operand in `p`'s symbol against the named target; it folds to a constant with no emitted code. `procedure(...), pointer :: fp` (#245 B3d) allocates a `ptr` alloca slot; `fp => my_func` stores the function address into it; a call through `fp` loads the slot and dispatches via an indirect `LR_OP_CALL`. Array pointers and conditional re-pointing across branches are not yet supported. |
| Polymorphism | An unlimited-polymorphic `class(*)` dummy is supported: a scalar actual (integer, real, or logical) is boxed into a `{void* data; i64 type_id}` descriptor passed by reference. `select type` on a `class(*)` selector (both `select type (v)` and `select type (a => v)`) dispatches on the runtime type id: each `type is (integer|real|logical)` arm runs when it matches with the associate name rebound to the concrete value, and `class default` runs when none match. `class(*)` locals, `class(t)` (typed polymorphic), derived-type guards, and `select rank` are unsupported. |
| Derived types | Simple program-scope derived type definitions with scalar integer, real (`real`, `real(real64)`), logical, and `c_ptr` components, plus fixed-size integer 1-D array components (`integer :: a(N)`, N a literal or integer parameter). Each component occupies whole 4-byte slots: `real(real64)` spans two, the others one. Real and logical components are scalar only. Scalar variables of those types are supported. Fixed-size arrays of derived-type instances (`type(t) :: arr(N)`) with element-component access (`arr(i)%comp`) are supported. A contained function may return a whole derived value (`q = make_point()`), passed via a hidden result-address argument. Component assignment, component reads, array-component element access (`x%a(i)`), `print`, and `stop` are supported. A derived value passes to a `type(t)` dummy by reference (the address of its component storage). A module-defined derived type resolves inside that module's procedures: a `type(t)` dummy (`intent(in)`/`out`/`inout`), local, or whole-derived result uses the same component layout as a program-scope type. A non-overridable type-bound binding (`procedure :: m => impl`) is callable as `obj%m(args)`, lowered to a direct `impl(obj, args)` with the object as the first argument (default `pass`); a `pass(name)` binding routes the object to the named dummy's position instead. Single inheritance (`type, extends(parent) :: child`) is supported: the child lays the parent's components first then its own (parent-first layout), so an inherited parent component is reachable on a child instance (`child%parent_comp`), an inherited array component keeps its element count, and a parent type-bound procedure is callable on a child instance through the inherited static binding. The parent must be defined before the extending type, and a child that declares its own type-bound procedures (override/extension) is not yet supported. Constructors, type parameters, overridable `class()` bindings with dynamic dispatch, typed polymorphic dummies (`class(t)`), nested derived types, whole-derived and whole-array-component assignment, character components, real and logical array components, and real/logical default initialisers are unsupported. |
| CLI | `-o <file>`, `-c`, `-I <dir>` (the include directories are searched for `.fmod` module artefacts on `use`), and positional `.o` link inputs. The source file may appear at any positional slot. Two-file separate compilation works: `ffc -c m.f90 -o m.o` then `ffc main.f90 m.o -o main`; the linked object's directory is also searched for its `.fmod`. Because module exports are compile-time constants and layout-only types, they fold into the program and the emitted executable is self-contained. |
| Module procedures | A module that `contains` procedures compiles when it is accompanied by a program unit: each module procedure is emitted under the mangled symbol `__<module>_MOD_<name>` (gfortran convention), so two modules may define a procedure of the same name without colliding, and calls through `use` resolve to the mangled symbol. Integer, real, and logical dummy and result types are supported. Character dummy and result types use the same ABI as contained procedures. A module procedure that itself `contains` an internal procedure is lowered: the internal procedures are emitted as flat functions under their plain Fortran names and calls inside the containing procedure resolve to them, the same flat-emission model used for program-contained procedures (no host association of the containing procedure's local variables). |
| Modules | A module defining named constants, derived-type definitions, module-level integer variables, and procedures compiles. Constants fold at use sites; types are layout-only. Module integer variables are emitted as zero-initialised globals named `__<module>_MOD_<var>` (gfortran convention) and persist across calls; the `.fmod` artefact records them so cross-file separate compilation works. `ffc -c` on such a source writes a sibling `<modulename>.fmod` describing the exports (see `docs/RUNTIME_ABI.md` module artefact format). `use <module>` resolves against an in-unit module or, failing that, a `<module>.fmod` found on the `-I` search path or the current directory. An intrinsic module (`iso_c_binding`, `iso_fortran_env`, ...) is recognised without a `.fmod`; any other unresolved module is a diagnostic. Module accessibility is honored: a bare `private`/`public` sets the default, `private`/`public :: name` overrides per name, and the attribute form (`integer, private, parameter`) marks a single declaration; a private symbol is not exported through `use`. |
| Interfaces | An `interface` block declaring `bind(c)` integer functions or `void` subroutines is parsed and registered (Fortran name, C name, argument and result kinds). Calls lower to a direct call on the C symbol with by-value arguments; an integer actual is passed as `i32`, and a `c_null_ptr` actual is passed as a null pointer (8 bytes). Non-`bind(c)`, generic, operator, and non-integer-result function interfaces are rejected. A contained procedure declared `bind(c, name="...")` is emitted under that unmangled C symbol (so it links from other languages), and internal calls resolve to the same symbol. |

## Unsupported categories

The full open issue list is on GitHub. Major categories tracked by long-
running tracker issues:

| #54 | Modules and separate compilation, including `.fmod` artefacts. Slices: #187-#190. |
| #55 | Fortran-aware scalar I/O runtime replacing the `printf` shim. |
| #56 | Generalised control-flow value merging (SELECT CASE fallthrough #180, derived-type IF merge #175, deferred-character IF merge #176). |
| #57 | Source locations and structured diagnostics. |
| #58 | FortFront compiler query boundary, replace arena reach-ins with public queries. |
| #59 | Standard- and Infer-mode conformance against the fortfront example corpus. |
| #167 | Self-hosting dependency map. |
| #173 | Upstream: FortFront compiler-API queries needed for self-hosting. |
| #174 | Upstream: LIRIC C-API capabilities needed for self-hosting. |

Unsupported features must fail during parsing, semantic analysis, or
lowering with a diagnostic. Silent partial lowering is a bug.

Generic specialization and cross-module inference wait for FortFront and
package-level orchestration contracts.

## Feature admission rule

A feature becomes supported only after all of these are true:

1. The FortFront API surface used by `ffc` is public and documented.
2. Any new runtime ABI is documented in `docs/RUNTIME_ABI.md`.
3. The direct LIRIC session lowerer implements the feature.
4. Behavioural tests cover success behaviour and relevant diagnostics.
5. The support contract, README, and development plan are updated in the same
   change.
