# ffc Support Contract

This document is the public contract for the active `ffc` compiler path.
A program is supported only when every construct it uses appears in
"Supported now". Everything else is unsupported until the linked issue is
closed with executable tests.

## Architecture boundary

- FortFront owns lexing, parsing, AST storage, semantic analysis, type
  inference, diagnostics, and the compiler-facing frontend query APIs.
- `ffc` owns lowering from FortFront semantic data to LIRIC, the runtime
  ABI, diagnostics produced during lowering, object emission, executable
  emission, and behavioural tests for every claimed feature.
- LIRIC owns native code generation behind its session C API.
- New `ffc` compiler work uses LIRIC session C APIs through
  `iso_c_binding`.
- New `ffc` compiler work does not use LLVM bindings, MLIR bindings,
  HLFIR, or text-IR compiler paths.

## Supported now

| Area | Supported contract |
| --- | --- |
| Input | Standard Fortran source parsed by FortFront's compiler-facing API. Lazy Fortran and Infer-mode inputs are accepted only when FortFront produces the same supported semantic constructs. |
| Output | Native executable or object file emitted by LIRIC. |
| Program unit | One main program plus contained procedures, or a file of one or more top-level modules with no main program. A program-less module file lowers to a no-op main: it compiles to an object with `-c` (each module's procedures under their mangled symbols) and links to an empty executable, matching gfortran's own module-only object. When one or more modules precede the program in a single file, FortFront wraps both under a multi-unit container; the program's own contained procedures are registered and lowered from inside that container, so a call to a program-contained scalar function resolves as contained rather than raising the unsupported-call diagnostic. |
| Scalars | Scalar `integer`, `real`, and `logical` declarations and assignments. Compound declarations (`integer :: a, b, c`). Non-default integer kinds `integer(1)`/`(2)`/`(8)` (also named via `int8`/`int16`/`int64` and the `iso_c_binding` C-interop kind names, including `c_size_t`/`c_intptr_t`/`c_ptrdiff_t`/`c_intmax_t` and the `c_int_least*`/`c_int_fast*` family) support declaration, assignment, `+`/`-`/`*`/`/` arithmetic, comparison, and `print`. `real(8)` accepts the conventional double-precision kind alias names `dp` and `wp` in addition to `real64`/`c_double`/`8`. `complex`/`complex(8)` scalars support declaration, `+`/`-`/`*`/`/` arithmetic (including a mixed real/complex operand, which promotes the real side with a zero imaginary part), `cmplx()`, `real()`/`aimag()` component extraction, and `print`. Scalar `parameter` named constants for `integer`, `real` (incl. `real(8)`/`real(dp)`), `logical`, and assumed-length `character(len=*)` with a compile-time literal initializer; the constant folds into uses in expressions, conditions, and `print`. Array, derived-type, and non-literal-expression parameters stay unsupported. The F2008 `TYPE(intrinsic-type-spec)` spelling (`type(integer)`, `type(real)`, `type(real(kind=4))`, `type(logical)`, `type(character(len=N))`, `type(complex)`, `type(double precision)`) names the intrinsic type itself and lowers exactly like the corresponding bare intrinsic declaration. A real literal's kind suffix naming another declared `integer, parameter` kind constant (`1.0_rk` where `integer, parameter :: rk = 8`) resolves that parameter's folded compile-time value instead of relying on a fixed alias list. Declaration-side `real(prec)`/`integer(prec)`/`complex(prec)` kind specs resolve the same way: any declared `integer, parameter` kind name (not just `dp`/`wp`) folds to its compile-time value and maps to the matching storage kind, on scalar declarations, `parameter` constants, derived-type components, dummy arguments, function results, and module variables. |
| Enums | Program-scope `enum, bind(c) ... end enum` blocks. Each `enumerator` binds an integer named constant of default kind: value is the explicit initializer (signed decimal literal), else the previous enumerator plus one, starting at zero. Multiple names per `enumerator` line and several enum blocks per program are supported; the constants fold into expressions, `print`, and `select case`. Module-scope enums and non-literal initializers stay unsupported. |
| Characters | Character literals, fixed-length `character(len=N)` variables, deferred-length `character(len=:), allocatable` with assignment, concatenation including self-aliasing and three-way concat, and compile-time `//` folding for character literal chains. A fixed-length target may also be assigned another character variable or a `trim`/`adjustl`/`adjustr`/`achar`/`char`/`repeat` result; the source is copied and blank-padded or truncated to the target's declared width, matching gfortran's fixed-length assignment. `==`, `/=`, `<`, `<=`, `>`, `>=` between character operands (variables, literals, `trim`/`adjustl`/`adjustr`/`achar`/`char`/`repeat` results, and `//` chains) use Fortran's blank-padded lexical ordering. `SELECT CASE` on a character selector accepts a lexical range label (`case ('a':'j')`) alongside literal labels. A character `parameter` named constant may declare a fixed length (`character(len=N), parameter`), which pads or truncates the folded initializer to N; its initializer may concatenate an earlier character named constant (`z = x // y`), not just literals. |
| Expressions | Integer and real literals, scalar variable references, integer and real arithmetic, integer comparisons (default `integer` and non-default `integer(1)`/`(2)`/`(8)`), scalar logical conditions. |
| Termination | Falling off `program main` returns zero. `stop <integer expression>` returns the integer expression as process status. |
| `if` | Integer comparison `if` blocks with terminating branches or branches that assign mergeable scalar values; scalar logical `if (flag)` conditions, including `.not.`/`.and.`/`.or.`/`.eqv.`/`.neqv.` trees, a logical array element (`if (flags(i))`), a derived-type logical component (`if (x%flag)`), `allocated(a)` on an allocatable array or deferred-length character, and a contained logical function's result. The one-line `if (cond) stmt` form without `then` works with any of these conditions. A character relational comparison (`==`, `/=`, `<`, `<=`, `>`, `>=`, including the dotted `.eq.`/`.ne.`/etc. spellings) uses Fortran blank-padded lexical ordering; either operand may be a character variable, literal, `//` concatenation, `trim()`/`adjustl()`/`adjustr()`/`repeat()`/`achar()`/`char()` result, a character array element (`arr(i)`), or a character-returning contained function call (`label(i) == "one"`). Array-element writes inside `if` branches are recognised even when the FortFront semantic flag is missing. A relational comparison (`==`, `/=`, `<`, `<=`, `>`, `>=`) whose two operands are of clearly incompatible intrinsic type classes (numeric versus character, or logical versus either) is rejected with a "mismatched types" diagnostic when both operand types are statically obvious (a literal or a declared scalar); an operand whose type is not statically resolvable leaves the comparison to the normal lowering path. |
| `do` | Counted `do` loops with integer bounds and literal integer step (positive or negative). |
| `goto` / labels | Simple `goto <label>` to a statement label in the same main program, including the arithmetic `if (e) l1, l2, l3` form that FortFront desugars to labelled `goto`s. A body that carries any statement label is lowered label-aware: each labelled statement starts a basic block, a `goto` branches to it, and every scalar integer/logical local is held in a stack slot so its value survives arbitrary branch edges (the backend rebuilds SSA). Computed `goto (l1, l2), e` and goto across a contained procedure boundary are not supported. |
| `pause` | `pause` and `pause <message>` continue execution. The interactive halt and stderr banner that gfortran emits are not reproduced. |
| `select case` | Single-arm or multi-arm `SELECT CASE` with literal integer labels, integer or character ranges (`case (1:5)`, `case ('a':'j')`), character labels, multi-label arms (`case (1, 5)`), and `case default`. Arms may terminate (`stop`/`return`) or fall through to a merge after the construct; non-terminated arms carry mergeable scalar values via an N-way PHI. An integer `SELECT CASE` whose CASE labels overlap is rejected with a diagnostic when the overlapping bounds are integer literals (single values, closed ranges, and open-ended ranges are compared; non-constant labels are skipped). |
| `select rank` | `SELECT RANK (x)` on a selector whose rank is statically known (a scalar is rank 0, a declared array carries its declared rank). The matching `rank (n)` arm is lowered inline, else `rank default`; arms that cannot match the static rank are dropped at compile time. Genuine assumed-rank `(..)` dummies, which need a runtime rank descriptor, and `rank (*)` are rejected with a diagnostic. |
| `where` | Single-statement `where (mask) a = expr` and the construct form with a final unmasked `elsewhere`, over fixed-size rank-1 integer, `real` (kind 4), and `real(8)` arrays. The mask is a scalar comparison whose operands are conforming array elements or broadcast scalars (`a > 0`, `b == 0`); integer masks use `==`, `/=`, `<`, `<=`, `>`, `>=`, real masks use `<`, `<=`, `>`, `>=`. Each body assignment lowers per element: the mask element selects between the body value and the current element (the `elsewhere` body selects the complement). Body right-hand sides accept whole-array `+`, `-`, `*`, scalar broadcast, and the elemental real intrinsics that lower to libm (`sqrt`, `exp`, `log`, `sin`, `cos`, `tan`, `abs`). Masked `elsewhere`, nested `where`, and array-section operands are rejected with a diagnostic. |
| `forall` | Single-statement `forall (i = lo:hi[:stride]) a(i) = expr` and the block form `forall (...) ... end forall`, over integer loop indices. Each index lowers to a counted loop; the indices form a sequential loop nest, so a multi-index `forall (i = ..., j = ...)` and a `forall` body that itself contains a `forall` both lower as nested loops. An optional trailing mask (`forall (i = lo:hi, i > 2) ...`) guards the body with a conditional branch; the mask is a scalar comparison over the loop indices and broadcast scalars. Body assignments write rank-1 or rank-2 array elements directly to memory. Masks whose operands are array elements (`a(i) > 0`) are rejected with a diagnostic. |
| `associate` | `associate (name => expr)` over scalar integer, real, or logical selectors, with one or several associations in one construct (`associate (u => a, v => b)`). Each name binds to the selector value for the construct body; the binding is dropped at `end associate`. The body may read the name in expressions, `print`, `stop`, and assignments. A rank-1 unit-stride array-section selector (`associate (x => a(lo:hi))`, source element kind integer, real(4), or real(8)) binds the name to a view onto the section's own storage, reindexed to lower bound 1: `size(x)`, element read, and element write all flow through to `a`. A scalar derived-type component selector (`associate (s => a%comp)`, component kind integer, real, or logical) binds the name to the component's own storage address: reads and writes through `s` flow to `a%comp`. Non-unit-stride and rank-2 array-section selectors, array-expression selectors (`z => x + y`), and character selectors are not yet supported. |
| Procedures | Contained integer, real, and logical functions and subroutines, including early `return` inside each. A function whose result type is left to Fortran implicit typing (no explicit result type on the header and no body declaration of the result variable) takes the default letter-rule kind: a result name beginning `i` through `n` is integer, otherwise real. An in-scope `implicit` statement that overrides the default is not honoured for the result kind. `stop` inside a contained procedure terminates the program via libc `exit`. Scalar procedure actual arguments use pointer parameters with copy-back for variable actuals. `optional` integer dummies are supported with `present(arg)`: an omitted actual is passed as a null pointer (the caller pads to the callee's parameter count) and `present` is a null check. `if (...)` without an `else` and `.not.` in a condition are supported. A `character(len=*), intent(in)` dummy receives a character actual (deferred-length variable or literal) through a {data pointer, length} descriptor, so `len`/`len_trim` see the actual length. A fixed-length `character(len=N), intent(in)` dummy uses the same descriptor but keeps its own declared width N rather than the caller's runtime length, so it sees exactly the first N bytes of a longer actual. A `character(len=:), allocatable, intent(out)` dummy receives the address of the actual variable's own descriptor, so an assignment inside the callee writes the new data and length back to the caller. A scalar actual (a scalar literal or an identifier bound to a non-array symbol) passed where the callee declares an array dummy is rejected with a `Rank mismatch in argument` diagnostic (F2018 15.5.2.4); the callee must be a procedure defined or interfaced in the same unit (an external call without an explicit interface is left alone), and an array element or section actual (a sequence-associated scalar such as `b(3)`) is not flagged. A call passing more actual arguments than that same in-unit callee declares dummies is rejected with a `More actual than formal arguments` diagnostic; fewer actuals than dummies is left to the optional-padding path (a trailing omitted optional dummy stays valid), and a type-bound call carrying a hidden receiver is not count-checked. |

| `print` | Scalar `print *, expr` for integer expressions, real values, character literals, character variables, logical literals, real variables, logical variables, and scalar `logical`-valued function calls (contained or module), which print as `T`/`F`. A bare whole `integer`/`real` array may appear as a print item alongside others (`print *, "tag", arr`); each element prints inline with list-directed spacing. An inline array constructor may also appear as a print item (`print *, [e1, e2, ...]`, alone or alongside other items): each explicit element (a literal, variable, or scalar function call of any supported scalar kind the underlying scalar-print path handles) is emitted like a scalar item with list-directed spacing. Implied-do (`[(i, i=1,n)]`) and character array constructors in print position stay unsupported. List-directed output matches gfortran byte-for-byte, including `real(8)` G-format (fixed and exponential forms, `Infinity`/`NaN`), `integer(4)` width, `T`/`F` logicals, and the inter-item separator rules. Every Fortran `real` is lowered as `real(8)`. Formatted `print fmt, items` with a literal format string is supported for a single `I`/`A` edit descriptor (`I0`, `Iw`, `A`, `Aw`) applied to each item, and for compound literal formats made from `I`, `X`, `F`, and `A` descriptors on stdout (`I5,2X,F8.3`, `A,A,I5`, `A8,I3`). The `A`/`Aw` descriptor in a compound format prints a character literal, fixed-length character variable, or deferred-length character expression. Other compound descriptors are rejected. Internal `write (buf, fmt) value` into a fixed-length character variable is supported for a single edit descriptor (`I0`/`Iw`/`A`/`Aw`, blank-padded to the buffer length) and for a compound literal format with one `I`/`A` descriptor per value in order (`write (buf, '(A,I0)') s, n`), each value formatted and concatenated in sequence before the final blank-pad/truncate to the buffer length; repeat counts and grouped descriptors in an internal write are not yet supported. Internal `read (buf, fmt) value` parses a single integer (`I0`/`Iw`) from a character variable via `sscanf` with a literal format, or, with a list-directed format (`read (buf, *) value`), a single integer, real, or fixed-length character scalar (a character token is read up to the next blank via `sscanf("%Ns", ...)`, then blank-padded/truncated to the target's declared length); no read-from-file-unit through this path. A `namelist /group/ a, b, c` declaration records the group's member list and emits no code; `write (unit, nml=group)` or `write (*, nml=group)` emits the `&GROUP` banner, one ` NAME= value,` line per scalar `integer`, `real`, or `logical` member (loaded from the symbol; logicals print `T`/`F`), and a closing ` /` line, to a file unit (`fprintf`) or stdout (`printf`). Array, character, and derived members, and namelist input (`read (unit, nml=group)`), are rejected with a diagnostic. |
| File I/O | `open(unit=u \| newunit=u, file=path[, status=...])` maps to `fopen`: `status='old'` opens for update (`r+`); `status='new'`/`'replace'` opens for update, truncating (`w+`); no `status=` (or `status='unknown'`) probes with `r+` first and falls back to `w+` only when the file does not already exist, matching gfortran's default `STATUS='UNKNOWN'` (an existing file's content is preserved, not truncated); a bare `open(u)` with no `file=` opens a scratch file (`tmpfile`). `close(u)` is a guarded `fclose` (a no-op on an already-closed unit); `rewind(u)` calls `rewind`. `write(u, *) items` and a literal-format `write(u, fmt) items` write list-directed/edit-descriptor output to the unit (see the `print` row's descriptor coverage) but not character items. `read(u, *) items` and a numeric-edit-descriptor `read(u, fmt) items` read whitespace-delimited tokens via `fscanf`, for integer, real, and fixed-length character scalar targets (a character token reads up to the next blank, then blank-pads/truncates to the declared length); `iostat=` reports 0/`iostat_end` from `feof`, and `end=<label>` transfers control on end-of-file inside a labeled program body. A numeric unit used without `open` is implicitly preconnected to `fort.<N>`. `inquire(file=path, exist=e[, iostat=i])` probes the filesystem with a guarded `fopen(path, "r")`/`fclose`; `inquire(unit=u, opened=o[, exist=e][, iostat=i])` reports whether `u` was opened earlier in the same straight-line control-flow path (a compile-time flag, not a runtime connection table) and always reports `exist=.true.`. `inquire(..., size=...)`, a formatted (non-list-directed) character/logical read (an `A`/`L` edit descriptor reads a fixed-width span rather than a whitespace token, which this fscanf-based reader does not implement), `advance='no'`, and namelist file I/O are not yet supported. A character-valued I/O specifier (`STATUS=`, `ACCESS=`, `ACTION=`, `FORM=`, `POSITION=`, `BLANK=`, `DELIM=`, `PAD=`, `ADVANCE=`, `ASYNCHRONOUS=`, `DECIMAL=`, `ROUND=`, `SIGN=`, `ENCODING=`, `IOMSG=`, ...) on `OPEN`/`CLOSE`/`INQUIRE`/`REWIND`/`READ`/`WRITE` is rejected with a "must be of type CHARACTER" diagnostic when given a numeric or logical literal value (`status=1`, `advance=5.`, `access=.false.`). |
| Arrays | Fixed-size arrays of rank 1 through 7 with compile-time bounds for `integer`, `real` (kind 4), `real(8)`, and `logical` element types. Logical arrays occupy `integer(4)` slots; element write/read, scalar broadcast, whole-array copy, and `print` (`T`/`F`) work for rank-1 and rank-2. Scalar integer parameters and explicit `lower:upper` bounds are supported. Rank-1 arrays support element assignment, element reads, `print`, `stop`, counted-loop subscripts, whole-array assignment from an array constructor (`a = [e1, e2, ...]`, element count must match the declared size), including typed constructors (`[integer :: ...]` truncates real literals toward zero, `[real :: ...]` promotes integer literals) and integer/real implied-do constructors (`[(i*i, i=1, n)]`, bounds and step fold at compile time), whole-array assignment and elemental `+`, `-`, `*`, `/`, `**` (real base with any real exponent via libm `pow`/`powf`; integer base with a non-negative integer literal exponent) between conforming arrays, an array constructor used as an rvalue operand alongside another array in the same expression (`a = b + [1, 2, 3]`), unary minus, and a general scalar expression (not just a literal or bare identifier, e.g. `a = real(i)`) broadcasting to every element, and scalar `size`, `sum`, `product`, `maxval`, `minval`, `dot_product`, `lbound(a, dim)`, `ubound(a, dim)`, `count(mask)`, `any(mask)`, `all(mask)`. Rank-1 and rank-2 array sections with compile-time integer bounds are accepted as rvalues in standalone list-directed `print` and whole-array copy/arithmetic when shapes conform. Rank-2 arrays support declaration and scalar element access through `a(i,j)` for reads, writes, `print`, `stop`, argument passing, whole-array copy, whole-array elemental `+`, `-`, `*` over conforming shapes, scalar `size`, `sum`, `product`, `maxval`, `minval`, `matmul`, `transpose`, and `reshape`; `shape(a)` can be assigned to a rank-1 integer array target. Arrays of rank 3 through 7 support declaration with compile-time bounds (literal or integer parameter, including explicit `lower:upper`), scalar element read/write via column-major addressing `a(i, j, k, ...)`, scalar broadcast, whole-array copy, whole-array elemental `+`, `-`, `*` over conforming shapes, whole-array `print`, `lbound(a, dim)`/`ubound(a, dim)`, `size(a[, dim])`, and `sum`; array sections, `matmul`, `transpose`, and `reshape` stay rank-1/rank-2 only. Runtime bounds checks are not emitted. An `integer/real/logical, allocatable :: a(:)` or `a(:,:)` declaration lowers to a zero-initialised 40-byte descriptor; `allocate(a(N))`/`allocate(a(M,N))` and `deallocate(a)` are supported (single variable, one or two dimensions, default lower bound). The element byte stride follows the kind: 4 for `integer`/`real(4)`/`logical`, 8 for `real(8)`. Scalar element access on an allocated 1-D or 2-D integer, real, or logical allocatable is supported for reads and writes (`a(i)`/`a(i,j)` as an rvalue and as an assignment target, including inside counted `do` loops), computing the address from the descriptor at runtime, and a logical element or whole array prints as `T`/`F`. Whole-array assignment from an array constructor (`a = [e1, e2, ...]`) is supported for 1-D integer, real, and logical allocatables: the old allocation is freed, fresh storage is allocated to match the constructor length, and elements are stored in order (auto-reallocation). Whole-array `print` (`print *, a`) is supported for 1-D allocatables when the most recent allocation extent is a compile-time constant; a runtime-only extent, or any rank-2 allocatable, leaves whole-array print unsupported. Whole-array assignment between two allocatables, and whole-array print or elementwise arithmetic on a rank-2 allocatable, are not yet supported. An assumed-shape dummy (`integer/real/logical, intent(...) :: a(:)` or `a(:,:)`) binds to the actual's base address; its extent is recovered from the caller's whole-array actual, which must have a compile-time size. The actual's declared bounds may name caller-scope integer `parameter` constants (`dimension(n)`), which fold from their declarations in the unit even when they are out of the callee's scope. This works both for procedures contained in a program and for module procedures. With that extent the dummy reuses the fixed-array machinery: element read and write, `size(a)`, `size(a, dim)`, `lbound(a, dim)`, `ubound(a, dim)`, `sum`, whole-array `print`, and rank-2 column-major addressing all work; inside a module procedure the element access is recognised from the symbol table even though FortFront leaves the usage node unflagged. An assumed-shape dummy whose actual is an array section, an array constructor, or an allocatable of runtime extent is rejected with a diagnostic (runtime-extent passing is not yet implemented). Fixed-size rank-1 and rank-2 `character(len=N)` arrays with a compile-time length and extent are supported: each element is stored as a pointer to a blank-padded string of the declared length. | Pointers | Scalar integer, real, and logical `pointer` and `target` (#245 slice B3a). A `target` local lives in a stable memory slot. Pointer assignment `p => t` makes the pointer address that slot; reads and writes through `p` dereference it, so a write through `p` mutates the target and a later write to the target is seen through `p`. `associated(p)` (one-argument form) and `associated(p, t)` (two-argument form, #245 B3c) report straight-line association state; `nullify(p)` disassociates. The two-argument form compares the compile-time address operand in `p`'s symbol against the named target; it folds to a constant with no emitted code. `procedure(...), pointer :: fp` (#245 B3d) allocates a `ptr` alloca slot; `fp => my_func` stores the function address into it; a call through `fp` loads the slot and dispatches via an indirect `LR_OP_CALL`. Rank-1 and rank-2 fixed-size integer, real, logical, and complex (`complex`/`complex(8)`) `pointer`/`target` arrays support whole-array `p => t` aliasing: the pointer adopts the target's base address, per-dimension shape, and element kind (for complex, both the real-part and imaginary-part component array bases), so element read/write, `lbound`/`ubound`, whole-array `print`, and `size` through `p` reach `t`'s storage. A local (non-dummy-argument) pointer/target array is supported; a pointer/target array declared as a dummy argument is not yet supported. | Polymorphism | An unlimited-polymorphic `class(*)` dummy is supported: a scalar actual (integer, real, or logical) is boxed into a `{void* data; i64 type_id}` descriptor passed by reference. `select type` on a `class(*)` selector (both `select type (v)` and `select type (a => v)`) dispatches on the runtime type id: each `type is (integer|real|logical)` arm runs when it matches with the associate name rebound to the concrete value, and `class default` runs when none match. `class(*)` locals, `class(t)` (typed polymorphic), derived-type guards, and assumed-rank `select rank` (see the `select rank` row) are unsupported. |
| Derived types | Simple program-scope derived type definitions with scalar integer, real (`real`, `real(real64)`), logical, and `c_ptr` components, plus fixed-size rank-1 integer, real, and logical array components (`integer :: a(N)`, `real :: r(N)`, `logical :: f(N)`, N a literal or integer parameter). Each array element occupies one i32 slot, except a `real(real64)` element which spans two, so element addressing scales the index by the slot width. Scalar variables of those types are supported. Fixed-size arrays of derived-type instances (`type(t) :: arr(N)`) with element-component access (`arr(i)%comp`) are supported. A contained function may return a whole derived value (`q = make_point()`), passed via a hidden result-address argument. Component assignment, component reads, array-component element access (`x%a(i)`), `print`, and `stop` are supported. A scalar nested derived component (`type(inner) :: c` inside an outer type) lays the inner type's slots inline and is accessed to any depth through its leaf scalar (`x%c%field`, `x%mid%inner%value`); the inner type may itself carry array components reached as a whole instance, but indexing an array component through a nested base (`x%c%v(i)`) is not yet supported. Default initialisation propagates through nested components: an inner type's own scalar component defaults materialise in an outer instance (`x%c%field` reads the inner default, not zero), and a nested component may carry a bare inner-type default-constructor initialiser (`type(inner) :: c = inner()`), which is equivalent to that default initialisation; an argument-bearing nested initialiser stays unsupported. A bare argumentless structure constructor (`v = t()`, or a `type(t) :: v = t()` initialiser) applies the type's default initialisation, including for a type with nested components that the argument-storing constructor path cannot build. A derived value passes to a `type(t)` dummy by reference (the address of its component storage). A module-defined derived type resolves inside that module's procedures: a `type(t)` dummy (`intent(in)`/`out`/`inout`), local, or whole-derived result uses the same component layout as a program-scope type. A non-overridable type-bound binding (`procedure :: m => impl`) is callable as `obj%m(args)`, lowered to a direct `impl(obj, args)` with the object as the first argument (default `pass`); a `pass(name)` binding routes the object to the named dummy's position instead. A `final` binding is accepted and ignored: ffc does not model finalisation, so the finaliser never runs and a program that observes its side effect mismatches rather than mis-passing, but a type carrying one is otherwise usable. Single inheritance (`type, extends(parent) :: child`) is supported: the child lays the parent's components first then its own (parent-first layout), so an inherited parent component is reachable on a child instance (`child%parent_comp`), an inherited array component keeps its element count, and a parent type-bound procedure is callable on a child instance through the inherited static binding. The parent must be defined before the extending type, and a child that declares its own type-bound procedures (override/extension) is not yet supported. A whole-derived scalar assignment copies one instance into another of the same type slot-by-slot (`y = x`). A scalar structure constructor whose components are all scalar integer, real, or logical stores its positional arguments into the target's slots (`x = t(1, 2.5, .true.)`); a component omitted from the constructor keeps its compile-time default, and inherited components construct in parent-then-child order. The same constructor also initialises a scalar derived variable at its declaration (`type(t) :: v = t(1, 2.5)`), materialised as an initialisation store into the fresh instance; a program-scope scalar derived `parameter` with a constructor initialiser (`type(t), parameter :: p = t(2, 3)`) lowers through the same initialised-instance path. A scalar `integer`, default-`real` (f32), or `logical` component default initialiser folds into the type's compile-time slot bytes and materialises when an instance is default-initialised. A constructor with a nested-derived, array, character, or `c_ptr` component argument, a keyword (`name=value`) argument, or whose type name is overloaded by a generic interface or contained function, falls back to the unsupported diagnostic rather than mis-constructing. Type parameters, overridable `class()` bindings with dynamic dispatch, typed polymorphic dummies (`class(t)`), allocatable and pointer derived components, arrays of derived components, whole-array-component assignment, indexing an array component through a nested base (`x%c%v(i)`), character components, and `real(real64)`/character/`c_ptr` default initialisers are unsupported. |
| CLI | `-o <file>`, `-c`, `-I <dir>` (the include directories are searched for `.fmod` module artefacts on `use`), and positional `.o` link inputs. The source file may appear at any positional slot. Because module exports are compile-time constants and layout-only types, they fold into the program and the emitted executable is self-contained. |
| Module procedures | A module that `contains` procedures compiles either accompanied by a program unit or as a bare module-only unit (lowered to a no-main object): each module procedure is emitted under the mangled symbol `__<module>_MOD_<name>` (gfortran convention), so two modules may define a procedure of the same name without colliding, and calls through `use` resolve to the mangled symbol. A self- or sibling-call inside a bare module procedure resolves, so a recursive module function lowers. Integer, real, and logical dummy and result types are supported; a `logical`-valued module function called from a program in the same file resolves and prints as `T`/`F`. Character dummy types use the same descriptor ABI as contained procedures. | Submodules | A single source file holding `module m` (declaring module procedures through interface bodies), one or more `submodule (m) s` units that implement them, and a program that calls through `use m` compiles. Each submodule extends its parent module's scope; its procedures are emitted under the parent's mangled symbol `__<m>_MOD_<name>`, so a call site resolves to the interface the parent declared regardless of which submodule supplies the body. Both the restated form (`module function f(x) result(y)` with its signature and declarations repeated in the submodule) and the separate form (`module procedure f` with the body only) are supported: the separate form inherits the dummy declarations, result variable, and function-versus-subroutine kind from the parent interface. A submodule whose parent module is absent from the compilation set is rejected with `submodule parent module not found`. Separate-file submodules (parent module in another `.f90`/`.fmod`) and nested submodule chains (`submodule (m:s) s2`) are not yet supported. |
| Modules | A module defining named constants, derived-type definitions, module-level scalar variables, and procedures compiles. Constants fold at use sites; types are layout-only. Module integer, real, and logical scalar variables are emitted as zero-initialised globals named `__<module>_MOD_<var>` (gfortran convention) and persist across calls; the `.fmod` artefact records each variable's kind so cross-file separate compilation loads and stores it at the right type. `ffc -c` on such a source writes a sibling `<modulename>.fmod` describing the exports (see `docs/RUNTIME_ABI.md` module artefact format); a source that defines several modules and no program compiles with `-c` to a single object with a `.fmod` per module beside it. `use <module>` resolves against an in-unit module or, failing that, a `<module>.fmod` found on the `-I` search path or the current directory. An intrinsic module (`iso_c_binding`, `iso_fortran_env`, ...) is recognised without a `.fmod`; any other unresolved module is a diagnostic. Module accessibility is honored: a bare `private`/`public` sets the default, `private`/`public :: name` overrides per name, and the attribute form (`integer, private, parameter`) marks a single declaration; a private symbol is not exported through `use`. A module-scope scalar variable of a registered derived type is emitted as a flat `[slots x i32]` global whose static bytes carry the type's compile-time component defaults (zero where a component has none); a using unit reads and writes its components through that global, including under a `use ..., alias => var` rename. An explicit structure-constructor initialiser (`type(t) :: v = t(2, 3)`) folds its compile-time integer arguments into the global's static slots, overriding the component defaults; an argument that is not a constant integer for an integer component keeps the initialiser unsupported. A scalar derived `parameter` initialised by such a constructor (`type(t), parameter :: p = t(2, 3)`) exports through the same slot global and binds as a derived reference in using units (honouring `use, only:`), rather than through the integer-only named-constant path. |
| Interfaces | An `interface` block declaring `bind(c)` integer functions or `void` subroutines is parsed and registered (Fortran name, C name, argument and result kinds). Calls lower to a direct call on the C symbol with by-value arguments; an integer actual is passed as `i32`, and a `c_null_ptr` actual is passed as a null pointer (8 bytes). Non-`bind(c)`, generic, operator, and non-integer-result function interfaces are rejected. A contained procedure declared `bind(c, name="...")` is emitted under that unmangled C symbol (so it links from other languages), and internal calls resolve to the same symbol. A named generic interface whose two specific procedures have indistinguishable dummy signatures (equal argument count with every position a known-kind scalar of identical type and kind) is rejected with an `ambiguous interfaces` diagnostic (F2018 C1514); a pair that differs in any position's type, kind, or array rank, or any position whose type/kind cannot be resolved statically, is left to normal lowering so a valid generic is never rejected. |

## Feature admission rule

A feature becomes supported only after all of these are true:

1. The FortFront API surface used by `ffc` is public and documented.
2. Any new runtime ABI is documented in `docs/RUNTIME_ABI.md`.
3. The direct LIRIC session lowerer implements the feature.
4. Behavioural tests cover success behaviour and relevant diagnostics.
5. The support contract, README, and development plan are updated in the same
   change.
